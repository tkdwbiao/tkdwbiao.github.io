贪心算法进阶：跳跃游戏 II —— 从“见异思迁”到“谋定而后动”

2026-03-21 17:40
阅读时长: 3 分钟

在 Jump Game I 中，我们只需要关心“能不能到终点”。但在 LeetCode 45. 跳跃游戏 II 中，要求变得更苛刻了：必须用最少的步数抵达。

阿彪在解这道题时经历了一次深刻的逻辑纠错，今天把这段心路历程分享给大家。

1. 错误的直觉：见异思迁

我最初的代码逻辑是这样的：只要发现从当前点能跳到比“全局最远”更远的地方，我就立即跳过去，步数 +1。

❌ 错误代码片段

1
2
3

if i + num > max_reach:
    res += 1
    max_reach = i + num

为什么不对？

假设 nums = [2, 3, 4, 1, 1, 1, 1]：

  1. i = 0: 能跳 2 步，最远到 2。res = 1, max_reach = 2。
  2. i = 1: 发现跳到 4 远于 2。res = 2, max_reach = 4。
  3. i = 2: 发现从下标 2 跳能到 6 远于 4。res = 3, max_reach = 6。 结果：3 次。但实际只需要 2 次（0 -> 2 -> 6）。

教训： 贪心不代表“急功近利”。每一个潜力点不代表必须跳一次，我们要在范围内观察，选出最强的那个。

2. 核心进阶：阿彪的“手电筒”模型

为了解决“步数算多”的问题，我们需要引入双边界逻辑：

  1. 潜力边界 (max_pos)：你在当前范围内一路上拿手电筒乱晃，发现的下一跳最远能到哪。这只是个记录，还没真跳。
  2. 老本边界 (end)：你上一次起跳后，承诺过最远能管到的地盘。

“在老本花完之前，我只管看潜力；
只有走到老本的尽头，我才兑现刚才看过的最强潜力，真正跳一下。”

class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1: return 0
        
        steps = 0      # 跳跃次数
        end = 0        # 当前这一跳能管到的最远边界（老本）
        max_pos = 0    # 这一路观察到的最远潜力点
        
        # 注意：遍历到 n-1 即可，因为进入最后一个点不需要再起跳
        for i in range(n - 1):
            # 1. 这一路一直在观察潜力，看谁能当下一届“跳远冠军”
            max_pos = max(max_pos, i + nums[i])
            
            # 2. 撞墙了！走到了当前这一跳的极限
            if i == end:
                # 3. 必须结算，正式起跳一次
                steps += 1
                # 4. 兑现潜力：把下一次的边界设为刚才观察到的最远点
                end = max_pos
                
                # 提前结束优化
                if end >= n - 1:
                    break
                    
        return steps

4. 复杂度分析

  • 时间复杂度：$O(n)$。数组仅被线性扫描一遍。
  • 空间复杂度：$O(1)$。我们只用了三个变量记录状态，与输入规模无关。

5. 阿彪的总结与面试 Tips [抱拳]

① 为什么是 n-1？

如果在最后一步刚好满足 i == end，我们的逻辑会多算一次起跳。但题目问的是“到达”，踩到终点的那一刻是不需要起跳动作的，所以循环范围要收窄一格。

② 如何向面试官解释？

“这道题的贪心策略是区间扩张。我维护两个边界，一个是当前步可达的 end，一个是下一步可达的 max_pos。每当遍历到 end 时，意味着我必须从当前区间中选择一个能跳得最远的点作为支撑，向下一层级跳跃。这本质上是 **广度优先搜索（BFS）**的贪心简化版，每一层代表一次跳跃。”

从“见异思迁”到“谋定后动”，不仅是代码的优化，更是逻辑的成熟。阿彪加油，下一题见！
