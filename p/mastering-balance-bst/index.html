二叉树重塑：从失衡到极致平衡的艺术

2026-03-19 19:05
阅读时长: 3 分钟

当一个二叉搜索树（BST）退化成一条链表时，它的查询效率会从 $O(\log N)$ 坠落到 $O(N)$。如何拯救它？LeetCode 1382. 将二叉搜索树变平衡 给出了一套完美的“手术方案”。

1. 第一步：利用“中序遍历”降维

在 BST 中，有一个至高无上的特性：

“BST 的中序遍历（左 -> 根 -> 右）必然得到一个升序数组。”

通过中序遍历，我们将复杂的树形结构“降维”成了一个有序数组。这是后续所有平衡化操作的基础。

2. 第二步：为什么“二分”建树就是平衡的？

拿到了有序数组 [1, 2, 3, 4, 5, 6, 7]，我们要把它重新构建成一棵树。为什么每次取 中点（mid） 作为根节点，建出来的树就是平衡的？

核心原理：

  1. 节点均匀分布：二分法每次选取 mid，保证了左边剩下的元素数量和右边剩下的元素数量差值绝对值不超过 1。
  2. 高度控制：树的高度取决于路径上节点的分叉情况。既然左右两边的“弹药（节点数）”是均等的，它们长出的高度也必然是几乎相等的。
  3. 数学直觉：
    • 数组长度为 $N$。
    • 左子树分配 $N/2$ 个节点，右子树分配 $N/2$ 个节点。
    • 递归下去，这棵树会像一张摊开的扇子一样，长得极其均匀，达到绝对平衡。

3. 第三步：建树中的“先序遍历”思想

虽然我们用的是有序数组，但建树的逻辑其实是“先序遍历”的逆过程。

  • 先序思想：根 -> 左 -> 右。
  • 建树逻辑：
    1. 算出 mid，立刻创建根节点 TreeNode(inorderSeq[mid])。（这是“根”）
    2. 递归构建左区间，接在 node.left 上。（这是“左”）
    3. 递归构建右区间，接在 node.right 上。（这是“右”）

这就是典型的**自顶向下（Top-down）**的构建方式。

4. 深度拆解：二分查找的写法与思想

这道题里的 build(left, right) 函数，本质上就是二分查找（Binary Search）的思想变体。

二分查找的核心模板：

  1. 区间定义：通过 left 和 right 锁死当前处理的范围。
  2. 中点定位：mid = (left + right) // 2。
  3. 递归/缩小规模：
    • 在查找中，我们根据大小关系跳向一边。
    • 在建树中，我们两边都要：左边用来盖左楼，右边用来盖右楼。

class Solution:
    def balanceBST(self, root: TreeNode) -> TreeNode:
        # 1. 中序遍历获取升序序列
        inorderSeq = []
        def getInorder(node):
            if not node: return
            getInorder(node.left)
            inorderSeq.append(node.val)
            getInorder(node.right)
        getInorder(root)

        # 2. 基于二分分治的思想重建树
        def build(left, right):
            # 终止条件：区间无效
            if left > right: return None
            
            # 选取区间中点，确保平衡
            mid = (left + right) // 2
            node = TreeNode(inorderSeq[mid])
            
            # 递归分治
            node.left = build(left, mid - 1)
            node.right = build(mid + 1, right)
            
            # 重要：必须返回节点，供上层父节点连接
            return node

        return build(0, len(inorderSeq) - 1)

5. 阿彪的面试避坑笔记 [抱拳]

  1. 变量名不要写串：在递归内部记得用 node.left 而不是外部的 root.left。
  2. 返回值不能漏：build 函数最后必须 return node，否则父节点拿到的全是 None，整棵树就散架了。
  3. 复杂度意识：
    • 时间复杂度：$O(N)$。中序遍历一次，建树访问每个元素一次。
    • 空间复杂度：$O(N)$。需要数组存结果，递归栈深度 $O(\log N)$。

阿彪小结： 中序遍历是 BST 的“透视图”，二分法是构建结构的“平衡尺”。掌握了这两点，所有的树形转换题目都难不住你！
