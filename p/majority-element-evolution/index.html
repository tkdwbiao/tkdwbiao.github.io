算法之美：从排序到摩尔投票，寻找数组中的‘霸主’

2026-03-22 22:05
阅读时长: 3 分钟

在 LeetCode 169. 多数元素 中，题目定义“多数元素”为出现次数大于 $\lfloor n/2 \rfloor$ 的数。这意味着它在数组中占据了“绝对半数”以上。

今天阿彪带大家拆解这道题的四种解法，尤其是那个极其优雅的 $O(1)$ 空间算法。

1. 境界一：排序法（最强直觉）

阿彪的思想： 如果一个数超过了半数，那么把数组排好序后，中点位置（$n/2$）一定坐着这个数。无论它是最小值、最大值还是中间值，它那庞大的数量都会横跨过中点。

🚀 代码实现

class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        nums.sort()
        return nums[len(nums) // 2]
  • 时间复杂度：$O(n \log n)$。取决于排序速度。
  • 空间复杂度：$O(1)$ 或 $O(\log n)$。

2. 境界二：哈希表法（通用解法）

核心思想： 老老实实拿个记事本，记下每个数字出现的次数。

from collections import Counter

class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        counts = Counter(nums)
        # 返回出现次数最大的键
        return max(counts.keys(), key=counts.get)
  • 时间复杂度：$O(n)$。只需遍历一遍。
  • 空间复杂度：$O(n)$。需要存下最多 $n/2$ 个不同的数。

3. 境界三：摩尔投票法（极致优化 - $O(1)$ 空间）

这是本题的灵魂所在。我们可以把它想象成一场 “混战打擂台”：

  1. 基本法则：自爆。不同的兵种遇到，同归于尽；相同的兵种遇到，抱团取暖。
  2. 逻辑：
    • 我们维护一个候选人 candidate 和一个票数 count。
    • 遍历数组：
      • 如果 count == 0，说明目前的擂台上没人了，当前这个数就是新的 candidate。
      • 如果当前数等于 candidate，count += 1。
      • 如果当前数不等于 candidate，count -= 1（两人同归于尽）。
  3. 为什么有效？：因为多数元素占了半数以上，即使全场其他人联合起来跟它对拼，最后剩下来的也一定是它。

class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        count = 0
        candidate = None

        for num in nums:
            # 1. 擂台空了，产生新候选人
            if count == 0:
                candidate = num
            
            # 2. 投票：相同加 1，不同减 1
            count += (1 if num == candidate else -1)

        return candidate
  • 时间复杂度：$O(n)$。
  • 空间复杂度：$O(1)$。只用了两个变量！

4. 境界四：分治法（思维拓展）

核心思想： 如果 a 是原数组的众数，那么将数组分成两半，a 至少是其中一半的众数。

  • 逻辑：递归拆分，找到左右两边的众数，如果一致则返回；如果不一致，则回过头去数一下这两个数在当前区间谁出现的次数更多。
  • 复杂度：$O(n \log n)$。

阿彪的总结对比表 [抱拳]

解法时间复杂度空间复杂度阿彪点评
暴力法$O(n^2)$$O(1)$会超时，别用
排序法$O(n \log n)$$O(1)$代码最简，容易想到
哈希表$O(n)$$O(n)$稳健，不限“过半”条件也适用
摩尔投票$O(n)$$O(1)$面试终极答案，性能怪兽

阿彪的面试避坑笔记

  1. 先问前提：面试时可以先确认“是否一定存在众数？”如果题目不保证一定存在，摩尔投票法在结束后还需要多加一步遍历来验证最后剩下的那个候选人是否真的超过了半数。
  2. 理解“过半”：摩尔投票法的核心前提是 $> n/2$。如果一个数的频次只是第一，但没过半（比如 [1, 2, 3]），摩尔投票法会失效。
  3. 中点逻辑：排序后的 nums[n/2] 这种写法，既能处理 n 为奇数，也能处理 n 为偶数，是个很棒的小技巧。

摩尔投票法不仅是一个算法，更是一种‘消耗战’的数学艺术。掌握了它，你就掌握了 $O(1)$ 处理数据流的秘诀。阿彪加油！
