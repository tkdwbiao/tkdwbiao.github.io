链表中的浪漫相遇：双指针法攻克相交链表

2026-03-17 20:00
阅读时长: 3 分钟

160. 相交链表

给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。

图示两个链表在节点 c1 开始相交：

题目数据 保证 整个链式结构中不存在环。

注意，函数返回结果后，链表必须 保持其原始结构 。

自定义评测：

评测系统 的输入如下（你设计的程序 不适用 此输入）：

  • intersectVal - 相交的起始节点的值。如果不存在相交节点，这一值为 0
  • listA - 第一个链表
  • listB - 第二个链表
  • skipA - 在 listA 中（从头节点开始）跳到交叉节点的节点数
  • skipB - 在 listB 中（从头节点开始）跳到交叉节点的节点数

评测系统将根据这些输入创建链式数据结构，并将两个头节点 headA 和 headB 传递给你的程序。如果程序能够正确返回相交节点，那么你的解决方案将被 视作正确答案 。

在处理 LeetCode 第 160 题 相交链表 时，最优雅的解法莫过于双指针法。它不仅代码简洁，更蕴含了一个巧妙的数学逻辑：走过彼此走过的路，终会相遇。

1. 核心思路：换道行驶

想象一下，你和女友分别在两条不同的路（链表 A 和 链表 B）上散步。这两条路在某个点汇合成了一条。

  1. 指针 p 从 A 的起点出发，走完 A 后，立即跳到 B 的起点开始走。
  2. 指针 q 从 B 的起点出发，走完 B 后，立即跳到 A 的起点开始走。

当这两个指针相等时（p is q），它们所指向的节点就是相交的起始节点。

2. 数学推理：为什么一定会相遇？

这是面试中面试官最关注的逻辑。我们通过一段简单的数学证明：

假设：

  • 链表 A 不相交的部分长度为 $a$
  • 链表 B 不相交的部分长度为 $b$
  • 两个链表相交的公共部分长度为 $c$

指针 p 的行进轨迹： 它先走完 A ($a + c$)，再走 B 的不相交部分 ($b$)。 总路程 = $(a + c) + b$

指针 q 的行进轨迹： 它先走完 B ($b + c$)，再走 A 的不相交部分 ($a$)。 总路程 = $(b + c) + a$

结论： 由于 $(a + c + b) = (b + c + a)$，两个指针走过的总路程是完全相等的。因此，它们一定会同时到达相交点。

3. 为什么不需要判断“不相交”的情况？

这是很多同学写代码时的疑问：如果两个链表根本不相交，这个 while 循环会不会死循环？

答案是：不会，它们最终会在 None 处相遇。

推理： 如果两个链表不相交，意味着公共部分长度 $c = 0$。

  • 指针 p 走完 A ($a$) 后换到 B，总路程达到 $a + b$ 时，它指向 None。
  • 指针 q 走完 B ($b$) 后换到 A，总路程达到 $b + a$ 时，它也指向 None。

此时，p 和 q 同时指向了 None。在 Python 中，None is None 为 True，while 循环结束，函数返回 p（即 None），这刚好符合题目要求。

这就是这段代码最优雅的地方：它利用 None 作为虚拟的“相交点”，完美统一了相交和不相交的逻辑。

class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        # 如果任何一个链表为空，则不可能相交
        if not headA or not headB:
            return None
        
        p, q = headA, headB
        
        # 当 p 和 q 相遇时（指向同一个节点或同时为 None）跳出循环
        while p is not q:
            # 如果走到了尽头，就换到另一条路的起点
            p = p.next if p else headB
            q = q.next if q else headA
            
        return p

5. 复杂度分析

  • 时间复杂度：$O(a + b)$。其中 $a, b$ 分别是两个链表的长度。每个指针最多遍历两个链表各一次。
  • 空间复杂度：$O(1)$。只使用了两个常数级别的指针变量。

阿彪的算法小结 [抱拳]

  1. 逻辑美感：双指针法消除了链表长度差带来的干扰。
  2. 代码技巧：利用 p = p.next if p else headB 这种简洁的写法，可以有效处理 None 的逻辑转换。
  3. 面试话术：如果面试官问起，就告诉他：“这个算法让两个指针抹平了长度差，使得它们同步进入了共同路径。”
