从 O(n²) 到 O(n)：深度拆解『和为 K 的子数组』

2026-03-17 18:00
阅读时长: 3 分钟

在算法面试中，如何处理“连续子数组”是一个高频考点。今天我们通过 LeetCode 560. 和为 K 的子数组，来看看一个算法是如何通过不断引入新的数据结构，实现从暴力到极致优化的。

1. 题目大意

给你一个整数数组 nums 和一个整数 k ，请你统计并返回该数组中和为 k 的子数组的个数。 注意：子数组是数组中连续的非空序列。

2. 三种境界的解法演进

第一种：暴力枚举（初级境界）

思路： 最直观的想法是找出所有的子数组 [i, j]，然后把中间的数加起来看等不等于 k。

  • 我们需要两层循环来确定 i 和 j。
  • 还需要一层循环来求和。

复杂度分析：

  • 时间复杂度：$O(n^3)$。如果数组很大，绝对会超时。
  • 空间复杂度：$O(1)$。

第二种：前缀和优化（中级境界）

思路： 在暴力法中，我们重复计算了大量的子数组和。如果能提前存下“前缀和”，就能在 $O(1)$ 的时间内算出任意子数组的和。

  • 前缀和定义：S[i] 表示从第 0 个元素到第 $i-1$ 个元素的总和。
  • 核心公式：子数组 [i, j] 的和 = S[j+1] - S[i]。

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        n = len(nums)
        s = [0] * (n + 1)
        # 预计算前缀和
        for i in range(n):
            s[i+1] = s[i] + nums[i]
            
        ans = 0
        # 两层循环枚举所有区间
        for j in range(1, n + 1):
            for i in range(j):
                if s[j] - s[i] == k:
                    ans += 1
        return ans

复杂度分析：

  • 时间复杂度：$O(n^2)$。虽然比 $O(n^3)$ 快，但面对 $10^5$ 级数据依然吃力。
  • 空间复杂度：$O(n)$。

第三种：前缀和 + 哈希表（终极境界）

思路： 这是面试官最想看到的 $O(n)$ 解法。我们将问题转化：站在位置 j，我们需要找左边有多少个 i 满足 S[j] - S[i] == k。 等式变形：S[i] = S[j] - k。

  • 我们不需要枚举 i，只需要在遍历 j 的同时，用哈希表记录下之前出现过的所有 S[i] 及其出现的次数。

from collections import defaultdict

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        ans = 0
        cnt = defaultdict(int)
        # 初始化：前缀和 0 出现过 1 次（处理从开头起和就为 k 的情况）
        cnt[0] = 1
        
        cur_s = 0 # 当前累加的前缀和
        for x in nums:
            cur_s += x
            target = cur_s - k
            
            # 1. 查表：左边有多少个位置能和我凑成 k？
            if target in cnt:
                ans += cnt[target]
            
            # 2. 存表：把当前前缀和存入哈希表，供后面使用
            cnt[cur_s] += 1
            
        return ans

复杂度分析：

  • 时间复杂度：$O(n)$。只遍历了一遍数组。
  • 空间复杂度：$O(n)$。哈希表最多存储 $n$ 个不同的前缀和。

3. 阿彪的深度思考：两个关键点

在写最优解时，这两个坑一定要避开：

① 为什么 cnt[0] = 1？

如果 nums = [3], k = 3，第一个数 cur_s 就是 3。此时 target = 3 - 3 = 0。如果不预设 cnt[0] = 1，我们就漏掉了这个从下标 0 开始的合法数组。它代表了 “和为 0 的前缀在最左侧虚拟出现过一次”。

② 为什么是 ans += cnt[target] 而不是 ans += 1？

因为数组中可能存在 0 或者 负数。这会导致不同的下标产生相同的前缀和。每一个相同的前缀和位置，都能与当前位置组成一个满足条件的区间。如果不累加次数，结果就会偏小。

4. 阿彪的算法小结

解法核心思想时间复杂度空间复杂度评价
暴力法遍历所有区间$O(n^3)$$O(1)$仅限理解思路
前缀和空间换时间$O(n^2)$$O(n)$基础进阶
哈希表优化目标差值检索$O(n)$$O(n)$面试满分答案

阿彪寄语：从 $O(n^2)$ 到 $O(n)$ 的跨越，本质是将“搜索”变成了“查表”。在数组题目中，当你发现需要反复查找某个数值时，第一时间想到哈希表，往往就是优化的突破口！

复习贴士：

  1. 记忆点：这道题不能用滑动窗口，因为数组里有负数。滑动窗口要求窗口移动具有单调性（即加一个数和一定变大），负数会破坏这个规则。
  2. 逻辑链：前缀和 $\rightarrow$ 固定右端点找左端点 $\rightarrow$ 哈希表记录左端点频次。
