Python 算法面试利器：深入详解 collections 模块

2026-03-15
阅读时长: 3 分钟

在准备算法面试（如 LeetCode）时，代码的简洁性和执行效率至关重要。Python 的内置容器（list, dict, tuple）虽然强大，但在处理特定逻辑（如分组、计数、双端队列）时，手动实现往往显得臃肿且易错。

Python 的 collections 标准库模块提供了一系列高性能的数据结构，熟练使用它们能让你的面试代码从“能跑”变成“优雅”。

1. defaultdict：告别 KeyError 检查

在处理哈希表（dict）时，最常见的逻辑是：如果键不存在，初始化一个默认值；如果存在，则更新它。

传统写法：

1
2
3
4
5
6

groups = {}
for word in strs:
    key = "".join(sorted(word))
    if key not in groups:
        groups[key] = []
    groups[key].append(word)

使用 defaultdict：

defaultdict 会在访问不存在的键时，自动调用构造函数（如 list, int, set）生成一个默认值。

1
2
3
4
5
6

from collections import defaultdict

groups = defaultdict(list)
for word in strs:
    key = "".join(sorted(word))
    groups[key].append(word)  # 无需判断 key 是否存在，直接 append

面试技巧：在处理“字母异位词分组”或“邻接表建图”时，这是首选。

2. Counter：计数器的终极方案

Counter 是 dict 的子类，专门用于统计元素出现的频次。

核心功能：

  • 自动计数：传入可迭代对象即可。
  • Top K 问题：most_common(k) 可以在 $O(N \log K)$ 时间内获取出现次数最多的元素。
  • 数学运算：支持加减法和交并集。

代码示例：

1
2
3
4
5
6
7
8

from collections import Counter

s = "leetcode"
count = Counter(s)
print(count)  # Counter({'e': 3, 'l': 1, 't': 1, 'c': 1, 'o': 1, 'd': 1})

# 获取出现次数最多的前 2 个元素
print(count.most_common(2)) # [('e', 3), ('l', 1)]

面试技巧：遇到“前 K 个高频元素”或“判断两个字符串是否互为字母异位词”时，直接祭出 Counter。

3. deque：高效的双端队列

在 Python 中，使用 list.pop(0) 的时间复杂度是 $O(N)$，因为需要移动所有后续元素。在实现 BFS（广度优先搜索） 时，这会导致超时。

deque（Double-ended queue）是为了在两端都能快速（$O(1)$）添加和弹出元素而设计的。

常用操作：

  • append() / appendleft()
  • pop() / popleft()

BFS 模板：

1
2
3
4
5
6
7

from collections import deque

queue = deque([root])
while queue:
    node = queue.popleft() # O(1) 操作
    if node.left: queue.append(node.left)
    if node.right: queue.append(node.right)

面试技巧：任何需要“先进先出（FIFO）”的场景，永远不要使用 list，请务必使用 deque。

4. namedtuple：让元组更具可读性

在算法中，我们经常用元组存储坐标 (x, y) 或区间 [start, end]。但通过索引访问 point[0] 可读性很差。

使用示例：

1
2
3
4
5
6
7

from collections import namedtuple

# 定义一个名为 Point 的结构，包含属性 x 和 y
Point = namedtuple('Point', ['x', 'y'])
p = Point(11, y=22)

print(p.x, p.y)  # 通过属性名访问，而不是索引 0 和 1

面试技巧：在处理复杂的 2D 矩阵问题或需要返回多个字段的函数时，namedtuple 能让你的代码像生产环境级别的代码。

总结

在算法面试中，合理使用 collections 模块有三大好处：

  1. 减少 Bug：减少了手动判断逻辑。
  2. 性能更优：底层由 C 实现，比 Python 手写循环快。
  3. 专业性：向面试官展示你对语言特性的深入理解。

建议背诵名单：

  • defaultdict(list)：用于分组。
  • Counter：用于计数。
  • deque：用于队列和 BFS。
