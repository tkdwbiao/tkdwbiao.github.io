算法利刃：深度拆解 Python 堆（Heap）的魔法

2026-03-20 14:00
阅读时长: 3 分钟

在处理“排行榜”、“前 K 个最大值”或者“中位数”问题时，排序虽然能解决，但往往太慢了。我们需要一种能实时动态维护极值的结构——这就是堆（Heap）。

1. 什么是堆？（物理直觉）

堆是一种特殊的完全二叉树：

  • 小顶堆 (Min-Heap)：父节点 $\le$ 左右子节点。堆顶永远是最小值。
  • 大顶堆 (Max-Heap)：父节点 $\ge$ 左右子节点。堆顶永远是最大值。

“堆的精髓：我不需要全场有序，我只要‘老板’（极值）永远坐在第一排。”

2. Python 里的 heapq：只有小顶堆

Python 提供了一个内置模块 heapq。注意：它默认只支持小顶堆！

import heapq

nums = [3, 1, 4, 1, 5, 9, 2]

# 1. 堆化：原地将列表转为堆 (时间 O(N))
heapq.heapify(nums)

# 2. 插入元素 (时间 O(log N))
heapq.heappush(nums, 0)

# 3. 弹出最小值 (时间 O(log N))
min_val = heapq.heappop(nums) # 返回 0

# 4. 只看一眼堆顶不弹出 (时间 O(1))
top = nums[0]

3. 阿彪的进阶：如何实现“大顶堆”？

因为 Python 没有内置大顶堆，程序员们通用的做法是：将所有数字取负号（* -1）入堆，取出来的时候再取负号转回来。

4. 终极实战：Top K 问题（面试必考）

题目：找出一个数组中最大的前 K 个数。

为什么用堆？

  • 排序法：$O(N \log N)$
  • 堆算法：$O(N \log K)$。当 $K \ll N$ 时（比如从 1 亿个数里找前 10 个），堆法快到起飞！

阿彪的“逆向思维”模版：

找最大的 K 个数，反而要维护一个“小顶堆”。

  1. 堆里只放 K 个数。
  2. 只要新来的数比堆顶（目前这 K 个里最菜的那个）大，就踢走堆顶，把新数放进去。
  3. 扫完一遍，堆里剩下的就是全场最强的 K 个。

def top_k_largest(nums, k):
    # 1. 取前 K 个建立小顶堆
    hp = nums[:k]
    heapq.heapify(hp)
    
    # 2. 遍历剩下的
    for i in range(k, len(nums)):
        if nums[i] > hp[0]:
            # 踢掉最小的，塞入当前大的
            heapq.heapreplace(hp, nums[i])
            
    return hp # 剩下的就是前 K 大

5. 阿彪的堆算法心得 [抱拳]

① 复杂度对比

操作时间复杂度
建堆 (Heapify)$O(N)$
插入 (Push)$O(\log N)$
删除 (Pop)$O(\log N)$
查询堆顶 (Peek)$O(1)$

② 面试金句

“在这道题中，虽然全局排序可以解决，但为了降低空间和时间复杂度，我选择了优先队列（堆）。通过维护一个固定大小为 K 的堆，我们可以将时间复杂度从 $O(N \log N)$ 优化到 $O(N \log K)$，这在处理海量数据流时极具优势。”

③ 常用函数避坑

  • heapq.nlargest(k, nums)：Python 封装好的直接找前 K 大，面试可以用，但手写堆逻辑更能展示功底。
  • 自定义对象入堆：如果入堆的是个对象（比如 (词频, 单词)），heapq 会默认比较第一个元素。

阿彪小结： 堆不是为了排序，而是为了动态寻找极值。记住“取反实现大顶堆”和“小顶堆找前 K 大”的逆向思维，堆这块你就通关了！

知识点的建议：

  1. 一定要练一遍“取反”逻辑：写大顶堆时，脑子里要有个弯：-10 > -20，所以原数较大的，取反后反而变小了，从而在小顶堆的顶部被保护起来（或者被优先处理）。
  2. Top K 题目：马上去 LeetCode 做一下 347. 前 K 个高频元素，这是堆和哈希表的完美结合。
  3. 理解 heapify 的 $O(N)$：面试官可能会问为什么建堆是 $O(N)$ 而不是 $O(N \log N)$（因为是从非叶子节点自底向上调整的），了解一下即可。
