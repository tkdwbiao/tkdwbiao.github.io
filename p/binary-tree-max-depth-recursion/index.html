递归的双面性：深度拆解二叉树最大深度的两种解法

2026-03-18 14:50
阅读时长: 2 分钟

二叉树的递归是 DFS 的大本营。在处理 LeetCode 第 104 题 二叉树的最大深度 时，我们可以从两个完全不同的方向去思考递归。

1. 核心思想：自底向上 vs 自顶向下

在递归中，信息的传递方向决定了你的解法流派：

  1. 自底向上 (Bottom-up)：先求子问题的解，再根据子问题的结果推导出父问题的解。
  2. 自顶向下 (Top-down)：将父节点的信息通过参数传递给子节点，在到达边界时记录答案。

2. 解法一：自底向上（分治思想）

“我要想知道我的深度，得先看两个孩子谁更深。”

这是最常用、也最简洁的解法。它利用了后序遍历的思想。

  • 逻辑：
    • 如果节点为空，深度为 0。
    • 递归求左子树深度。
    • 递归求右子树深度。
    • 当前节点深度 = max(左深, 右深) + 1。

class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        # Base Case: 到达空节点，深度为 0
        if root is None:
            return 0
            
        # 1. 递归求左、右子树深度
        l_depth = self.maxDepth(root.left)
        r_depth = self.maxDepth(root.right)
        
        # 2. 结果合并：当前深度 = 子树最大深度 + 1 (当前层)
        return max(l_depth, r_depth) + 1

3. 解法二：自顶向下（维护全局状态）

“我下楼时带着计步器，每走一层就记一下，最后看最大值。”

这利用了前序遍历的思想，将深度信息作为参数往下传。

  • 逻辑：
    • 进入一个节点，深度 +1。
    • 更新全局最大值 ans。
    • 继续带着新的深度去走左右子树。

class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        ans = 0
        
        def dfs(node, depth):
            if node is None:
                return
            
            # 1. 状态处理：每深入一层，深度加 1
            depth += 1
            
            # 2. 更新全局最大值
            nonlocal ans
            ans = max(ans, depth)
            
            # 3. 递归推进：把当前的深度告诉孩子们
            dfs(node.left, depth)
            dfs(node.right, depth)
            
        dfs(root, 0)
        return ans

4. 做题技巧：为什么“假设递归结果正确”很有用？

网友说：「在写递归函数时，可以假设递归返回的结果一定是正确的」。

灵神总结：这其实是数学归纳法在编程中的应用。

  • 不要试图去展开每一层递归（人脑模拟会死循环）。
  • 你只需要确保：
    1. Base Case（如 root is None）是正确的。
    2. 如果左右子树给我的结果是真的，我如何利用它们凑出我的结果。

5. 复杂度分析（面试必答）

  • 时间复杂度：$O(n)$。每个节点都被访问且仅访问一次。
  • 空间复杂度：$O(n)$。
    • 最坏情况：树退化成一条链（类似链表），递归栈深度为 $n$。
    • 最好情况：平衡二叉树，递归栈深度为 $\log n$。

阿彪小结 [抱拳]

  • 自底向上更符合函数式编程，代码更短。
  • 自顶向下更像是在搜索路径，适合需要统计“从根到某个点”信息的题目。

在面试中，如果你能同时给出这两种思路，并解释清楚 “深度信息如何在父子之间传递”，面试官对你的评价会直接拉满！
