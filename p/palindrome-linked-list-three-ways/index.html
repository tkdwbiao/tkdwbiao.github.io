从 O(n) 空间到 O(1) 极致攻克回文链表

2026-03-17 23:23
阅读时长: 3 分钟

判断数组是否回文很简单，但链表只有 next 指针，无法回头。如何在 $O(n)$ 时间内高效判断回文链表（LeetCode 234）？本文提供三套进阶解法。

解法一：数组转换法（最直观）

核心思想： 链表不支持随机访问，但数组支持。我们将链表的所有值拷贝到数组中，利用数组的双指针或切片轻松判断。

class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        # 1. 将链表数值全部存入数组
        vals = []
        cur = head
        while cur:
            vals.append(cur.val)
            cur = cur.next
        
        # 2. 利用 Python 切片反转特性对比，或者使用双指针前后向中间靠拢
        return vals == vals[::-1]
  • 复杂度分析：
    • 时间复杂度：$O(n)$，遍历一遍链表，一遍数组。
    • 空间复杂度：$O(n)$，需要额外数组存储所有节点值。
  • 评价：面试中如果不限空间，这是最稳、最快 AC 的解法。

解法二：递归回溯法（最优雅）

核心思想： 利用递归函数的调用栈。递归会先一直深入到最后一个节点，在函数“弹栈”返回时，实际上是在倒序遍历链表。我们同时维护一个全局的前序指针，实现“首尾对比”。

class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        self.left = head # 全局指针，用于从前往后走

        def traverse(right):
            if not right: return True
            
            # 1. 递归深入到最末尾
            res = traverse(right.next)
            
            # 2. 回溯过程：此时 right 是从后往前走的，self.left 是从前往后走的
            # 两者进行值比对
            res = res and (self.left.val == right.val)
            
            # 3. 前序指针向后移动一位
            self.left = self.left.next
            return res

        return traverse(head)
  • 复杂度分析：
    • 时间复杂度：$O(n)$。
    • 空间复杂度：$O(n)$，递归深度为 $n$，占用 $O(n)$ 的栈空间。
  • 评价：代码极其优雅，充分体现了对递归回溯本质的理解。

解法三：快慢指针 + 局部反转（极致优化）

核心思想： 为了达到 $O(1)$ 额外空间，我们必须原地操作：

  1. 找中点：快慢指针，快指针走两步，慢指针走一步。
  2. 反转：反转慢指针之后的后半部分链表。
  3. 比对：前半部分和反转后的后半部分进行双指针比对。

class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        if not head or not head.next: return True
        
        # 1. 快慢指针寻找中点
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
        # 2. 原地反转后半部分链表 (从 slow 开始)
        pre, cur = None, slow
        while cur:
            nxt = cur.next
            cur.next = pre # 反转指向
            pre = cur      # 指针后移
            cur = nxt
            
        # 3. 比对前后两段
        p1, p2 = head, pre # p1指向头，p2指向反转后的后半段头
        while p2:
            if p1.val != p2.val:
                return False
            p1 = p1.next
            p2 = p2.next
            
        return True
  • 复杂度分析：
    • 时间复杂度：$O(n)$。
    • 空间复杂度：$O(1)$，只用了常数个指针变量。
  • 评价：面试满分答案。虽然逻辑复杂，但证明了你对指针控制和空间优化的极致掌握。
