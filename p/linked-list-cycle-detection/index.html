猫鼠游戏：判定环形链表的三种境界

2026-03-18 17:00
阅读时长: 3 分钟

如何判断一个链表是否有环？（LeetCode 141）这个问题有三种解法，分别代表了对空间利用的三种理解：从耗费空间的“记账”，到破坏结构的“标记”，再到极致优雅的“奔跑”。

1. 题目来源

LeetCode 141. 环形链表

2. 三种解法深度拆解

解法一：哈希表法（空间换时间）

核心思想： 这是最符合直觉的方法。准备一个“黑名单”（哈希表 set），每走过一个节点就存进去。如果发现当前节点已经在黑名单里，说明我们“转回来了”。

class Solution:
    def hasCycle(self, head: ListNode) -> bool:
        seen = set()
        while head:
            if head in seen:
                return True
            seen.add(head)
            head = head.next
        return False
  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(n)$，需要存下所有节点的引用。

解法二：原地标记法

核心思想： 既然不想额外开辟 $O(n)$ 的集合，那我们能不能在路过节点时留个记号？ 阿彪的想法是：每经过一个节点，就把它的值改成一个特殊值（如 "0"）。 如果下次遇到的节点值已经是 "0"，说明有环。

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        while head:
            # 1. 如果遇到标记过的特殊值，说明绕回来了
            if head.val == "0":
                return True
            # 2. 原地标记：把走过的路封死
            head.val = "0"
            head = head.next
        return False
  • 阿彪点评：这一招非常犀利，空间复杂度降到了 $O(1)$（不需要额外数组或集合）。
  • 面试注意：这种做法被称为 “破坏性修改”。在真实工程中，修改原始数据可能导致其他功能崩溃。但在面试中，你可以主动跟面试官讨论：“如果允许修改原数据，我可以实现 $O(1)$ 的标记法。” 这种 成本意识是加分项！

解法三：快慢指针（乌龟与兔子的追逐战）

核心思想： 如果不允许修改原数据，还想实现 $O(1)$ 空间，那就祭出经典的 “龟兔赛跑算法”（Floyd 判圈算法）。

  1. 慢指针 (slow)：每次走 1 步。
  2. 快指针 (fast)：每次走 2 步。
  3. 逻辑：如果没有环，快指针会直接跑到 None；如果有环，快指针最终会在环内“套圈”撞上慢指针。

class Solution:
    def hasCycle(self, head: ListNode) -> bool:
        if not head or not head.next:
            return False
        
        slow = head
        fast = head.next # 初始位置错开，方便进入 while 循环

        while slow != fast:
            # 如果快指针到头了，说明没环
            if not fast or not fast.next:
                return False
            slow = slow.next
            fast = fast.next.next
        
        return True
  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(1)$，且不破坏原链表结构。

3. 阿彪的算法小结 [抱拳]

解法空间复杂度优点缺点
哈希表$O(n)$逻辑最简单，代码不易写错内存开销大
原地标记$O(1)$代码极致简洁，思路奇特破坏原数据结构
快慢指针$O(1)$算法最优雅，工业界标准解法逻辑证明稍复杂

阿彪做题心得： 在面试中，面对这一题，我建议先给出快慢指针解法。如果想展示自己的思路灵活，可以补充一句：“其实还可以通过原地修改节点值的方式来实现 $O(1)$ 空间，但那样会改变原始数据。”

这种多维度的思考，能让面试官感受到你不仅会刷题，还能根据实际场景权衡利弊！

阿彪复习贴士：

  1. 原地标记的小技巧：你可以把 head.val 改成一个极其特殊的对象（例如 object()），这样就不会和真实的数值（比如原本就有 0）撞车了。
  2. 快慢指针的间距：只要快慢指针的速度不一样，在有环的情况下一定会相遇。因为每走一步，快慢指针的距离就会缩小 1，最终距离归零即相遇。
