二叉树直径全解：从‘模块化设计’到‘极致 O(n) 优化’

2026-03-19 12:00
阅读时长: 3 分钟

二叉树的直径（LeetCode 543）是面试中的常客。今天阿彪融合了 LeetCode 网友的“工程化拆解”视角和常用的“极致性能”解法，带大家从三个维度彻底攻克这道题。

1. 核心物理直觉：寻找“拐点”

无论用什么方法，我们要锁死的物理本质只有一句话：

“任意节点作为‘拐点’，经过它的最长路径长度 = 左子树最大深度 + 右子树最大深度。”

全局最长路径不一定经过根节点（Root），但它一定会以树中的 **某一个** 节点作为最高点。

2. 视角一：模块化递归（工程美学版）

很多大厂（如京东、美团）在面试时会考察代码的 可读性。将“计算深度”和“遍历树”拆分成两个函数，逻辑最清晰。

class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.max_dia = 0

        # 功能一：专门求任意节点的最大深度
        def get_max_depth(node):
            if not node: return 0
            return max(get_max_depth(node.left), get_max_depth(node.right)) + 1

        # 功能二：专门负责走遍树的每一个节点
        def traverse(node):
            if not node: return
            # 计算以当前节点为拐点的直径
            self.max_dia = max(self.max_dia, get_max_depth(node.left) + get_max_depth(node.right))
            # 继续递归遍历左右子树
            traverse(node.left)
            traverse(node.right)

        traverse(root)
        return self.max_dia
  • 评价：代码极其符合直觉，易于维护。
  • 复杂度：$O(n^2)$。存在大量的重复深度计算。

3. 视角二：迭代遍历（显式栈控版）

面试官有时会追问：“如果不允许使用递归遍历整棵树，你怎么做？” 这时我们可以用 栈（Stack） 来模拟深度优先搜索，但计算深度时依然保持递归，这是一种“半迭代”的折中方案。

class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        if not root: return 0
        def get_max_depth(node):
            if not node: return 0
            return max(get_max_depth(node.left), get_max_depth(node.right)) + 1

        res = 0
        stack = [root]
        while stack:
            node = stack.pop()
            # 同样是计算拐点，只是遍历方式换成了迭代
            res = max(res, get_max_depth(node.left) + get_max_depth(node.right))
            if node.right: stack.append(node.right)
            if node.left: stack.append(node.left)
        return res

4. 视角三：自底向上（极致性能版 $O(n)$）

这是阿彪最初推荐的“王牌解法”。既然每一个节点在计算深度时，都需要知道左边多深、右边多深，那我们干脆 “在计算深度的过程中，顺手把直径算了”。

class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.ans = 0 

        def depth(node):
            if not node: return 0
            # 1. 拿到左右子树深度
            L = depth(node.left)
            R = depth(node.right)
            
            # 2. 核心动作：更新全局最大值（一箭双雕）
            self.ans = max(self.ans, L + R)
            
            # 3. 返回给父节点应有的深度
            return max(L, R) + 1

        depth(root)
        return self.ans
  • 评价：面试满分答案。每个节点仅访问一次，没有任何重复计算。
  • 复杂度：时间 $O(n)$，空间 $O(h)$。

5. 阿彪的横向对比与总结 [抱拳]

维度模块化递归迭代版自底向上 (最优)
可读性极佳一般较好
性能$O(n^2)$$O(n^2)$$O(n)$
适用场景初步理清逻辑考察栈的应用最终提交/正式面试

阿彪的面试话术建议：

“面试官，这道题的直观解法是拆分功能，遍历每个节点并计算其左右深度之和（$O(n^2)$）。但为了追求极致性能，我采用了后序遍历自底向上的策略。这样在一次 $O(n)$ 的遍历中，利用子问题的深度结果直接推导出当前节点的直径贡献，从而避免了重复计算。这体现了**动态规划（DP）**中‘重用子问题解’的思想。”
