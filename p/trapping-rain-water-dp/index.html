经典算法：接雨水问题的动态规划拆解

2026-03-17
阅读时长: 3 分钟

接雨水（Trapping Rain Water）是 LeetCode 第 42 题，也是一道经典的“高频硬骨头”。它的解法很多，包括双指针、单调栈等，但 动态规划（DP） 是最符合直觉、最能体现“空间换时间”思想的解法。

1. 核心物理原理：木桶效应

想象一下，如果你站在数组的某一个位置 $i$，你头顶上能存多少水？ 答案并不取决于当前的柱子有多高，而是取决于你左右两边 最高的两堵墙 ：

  1. 水位线：由左右两堵最高墙中 较矮 的那一堵决定（木桶效应）。
  2. 实际水量：水位线高度减去你脚下柱子本身的高度 height[i]。

计算公式：

$$第 i 格的水量 = \min(\max(\text{left}), \max(\text{right})) - \text{height}[i]$$

注：如果结果小于 0，则该格不存水。

2. 动态规划思路

如果我们对每个位置都暴力向左右扫描寻找最高点，时间复杂度会达到 $O(n^2)$。 为了优化，我们预先计算出两个数组：

  • left_max：记录每个位置 左侧（含自身） 出现的最高柱子高度。
  • right_max：记录每个位置 右侧（含自身） 出现的最高柱子高度。

class Solution:
    def trap(self, height: List[int]) -> int:
        if not height: return 0
        n = len(height)
        
        # 初始化两个 DP 数组
        left_max = [0] * n
        right_max = [0] * n
        
        # 从左向右扫描，计算 left_max
        left_max[0] = height[0]
        for i in range(1, n):
            left_max[i] = max(height[i], left_max[i-1])
            
        # 从右向左扫描，计算 right_max
        right_max[n-1] = height[n-1]
        for i in range(n-2, -1, -1):
            right_max[i] = max(height[i], right_max[i+1])
            
        # 计算总水量
        result = 0
        for i in range(n):
            # 核心公式应用
            result += min(left_max[i], right_max[i]) - height[i]
            
        return result

3. 复杂度分析

  • 时间复杂度：$O(n)$。我们经历了三次线性遍历（两次预处理数组，一次累加水量）。
  • 空间复杂度：$O(n)$。使用了两个长度为 $n$ 的数组来存储中间状态。

4. 面试进阶：能否将空间复杂度降到 $O(1)$？

面试官看到上面的代码后，通常会问：“你能不使用额外数组完成吗？”

答案是：双指针（Two Pointers）。 由于 left_max 是从左往右增大的，right_max 是从右往左增大的，我们可以使用两个指针 left 和 right 从两端向中间移动。

  • 维护两个变量 l_max 和 r_max。
  • 如果 l_max < r_max，说明左边的水位由 l_max 决定，处理左侧并移动 left。
  • 反之，处理右侧并移动 right。

def trap(height):
    left, right = 0, len(height) - 1
    l_max, r_max = 0, 0
    res = 0
    while left < right:
        l_max = max(l_max, height[left])
        r_max = max(r_max, height[right])
        if l_max < r_max:
            res += l_max - height[left]
            left += 1
        else:
            res += r_max - height[right]
            right -= 1
    return res

总结

  1. 动态规划解法是接雨水问题的底层逻辑，它告诉我们要关注每个柱子的“左右边界”。
  2. 双指针是动态规划的极致空间优化版，适合在大厂面试中作为最终的“大招”祭出。

复习建议：

  1. 记忆公式：面试时先说 min(left_max, right_max) - height。
  2. 对比差异：这道题和《最大矩形》或者《柱状图中最大的矩形》长得很像，但逻辑完全不同。接雨水看的是“凹槽”，矩形看的是“凸起”。
  3. 练习手感：动态规划版本最不容易写错，建议先把这个版本练熟，确保在面试时能一次性 Bug-free。
