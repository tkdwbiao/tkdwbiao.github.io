递归算法全解析：从三类核心题型到底层思维

2026-03-18 13:00
阅读时长: 5 分钟

在算法面试中，递归不是一种具体的算法，而是一种思维模式。它的本质在于将一个庞大的问题，拆解为若干个完全一致的子问题。

很多同学怕递归，是因为不知道从何下手。其实，递归题目在面试中无外乎三大“题型流派”，只要对号入座，便可迎刃而解！

1. 递归的三大核心题型

流派一：处理结构型 (Traverse & Modify)

特征：问题的处理依附于树或链表的物理结构。 思想：“我只管处理当前节点，剩下的扔给子节点去做。” 经典例题：

  • LeetCode 234. 回文链表（利用递归回溯实现倒序遍历）
  • LeetCode 206. 反转链表（递归版）

流派二：分治型 (Divide & Conquer)

特征：将大问题拆分为左右两个子问题，合并结果。 思想：子问题独立求解，最后通过合并算子（Operator）将结果汇聚。 经典例题：

  • LeetCode 104. 二叉树的最大深度 (深度 = 1 + max(左深, 右深))
  • LeetCode 226. 翻转二叉树 (分别翻转左右子树，再交换)

流派三：决策树/回溯型 (Backtracking)

特征：每一步都有多种选择，需要遍历所有可能路径。 思想：做选择 -> 递归进入 -> 撤销选择。 经典例题：

  • LeetCode 79. 单词搜索（从当前点向四个方向探索）
  • LeetCode 46. 全排列（排列组合的终极模板）

2. 阿彪递归三步公式

遇到递归题，在草稿纸上不要直接写代码，先写这三段：

  1. 确定出口 (Base Case)：

    • 什么时候问题不再可拆？（例如：到达叶子节点 not node，或到达字符串末尾 index == len(word)）
    • 思考：出口返回的是什么？（布尔值？0？还是 None？）

  2. 状态处理 (Logic)：

    • 当前层要做什么？（比对？加和？标记已访问？）

  3. 递归推进 (Recursive Call)：

    • 缩小问题规模的参数是什么？（node.next？index + 1？r+1？）

3. 递归的“三位一体”模板

写递归时，阿彪建议你永远按这三个步骤进行“填空式”思考：

  1. 终止条件 (Base Case)：什么时候停？（防止死循环，通常返回一个初始值）。
  2. 递归深入 (Recursive Call)：去解决规模更小的子问题。
  3. 结果合并 (Backtrack/Combine)：拿到了子问题的答案，我这一层要做什么收尾工作？

4. 递归三大核心题型与解法

第一类：校验类（逻辑：一票否决制）

核心逻辑：只要子递归发现不满足条件，立即向上传递 False。

  • 代表题型：234. 回文链表、101. 对称二叉树。
  • 技巧：Base Case 通常返回 True，因为“没发现问题就是没问题”。

def check(node):
    # 1. Base Case: 到底了也没发现错
    if not node: return True  
    
    # 2. 递归检查子节点（信号由下往上传递）
    if not check(node.next): 
        return False  # 子层报错，全盘皆输
        
    # 3. 本层判断逻辑
    return node.val == self.front.val

第二类：搜索类（逻辑：一票通过制）

核心逻辑：只要有一个方向找到了答案，立即停止搜索，向上传递 True。

  • 代表题型：79. 单词搜索、路径查找。
  • 技巧：利用 or 的短路特性，一旦找到就截断。

def dfs(r, c, index):
    # 1. Base Case: 找到了！
    if index == len(word): return True  
    
    # 2. 递归深入：尝试四个方向
    # 利用 or 的短路特性，只要一个返回 True，后面的就不跑了
    if dfs(r+1, c) or dfs(r-1, c) or dfs(r, c+1) or dfs(r, c-1):
        return True
        
    return False

第三类：计算/累加类（思想：分治合并）

核心逻辑：当前层的结果 = “当前值” 与 “子问题值” 的运算。

  • 代表题型：104. 二叉树的最大深度、206. 反转链表（递归版）。
  • 技巧：核心公式通常是 return current_val + solve(sub_problem)。

def get_depth(node):
    if not node: return 0
    # 结果 = 1 (当前层) + 左右子树中较深的一个
    return 1 + max(get_depth(node.left), get_depth(node.right))

5. 阿彪的做题技巧：如何不被递归“绕晕”？

① 拒绝人肉递归

千万不要试图在脑子里模拟递归的每一层跳转，那是计算机干的事，人脑模拟容易导致“栈溢出”。 正确做法：假设子函数已经帮你算好了正确答案，你只需要考虑 “拿到这个答案后，我这一层怎么收尾”。

② 关注“信号方向”

  • 自顶向下 (Top-down)：通过参数传递状态（如：dfs(..., index + 1)）。
  • 自底向上 (Bottom-up)：通过返回值（return）回传结果（如：回文判断的 True/False）。

③ 内存预警

递归是有成本的，每次调用都会产生一个栈帧。如果数据量达到 $10^5$，递归会报 RecursionError。 面试加分点：主动提一句：“如果数据量极大，我会考虑用显式栈（Stack）来模拟递归，或者改为迭代法。”

5. 面试实用技巧

  • 信号传递机制：

    • 校验类：使用“一票否决制”。子递归只要返回 False，父递归立即跟进 return False。
    • 搜索类：使用“一票通过制”。子递归只要返回 True，父递归立即终止后续遍历，直接 return True。

  • 时间与空间分析“加分话术”： 在分析递归复杂度时，面试官更喜欢听你这样说：

    “递归的时间复杂度通常取决于 $O(子问题个数 \times 单个子问题复杂度)$，空间复杂度主要取决于函数调用栈的深度，在最坏情况下为 $O(depth)$。我会注意递归深度过大的潜在溢出风险，并考虑必要时使用 sys.setrecursionlimit 优化或改写为迭代法。”

  • 代码调试建议： 如果你的递归写挂了，不要盯着总代码看！ 单独打印这三个东西：

    1. 传入参数（看看有没有随着规模缩小）
    2. 弹栈顺序（在回溯过程中 print 一下，看看顺序是不是你想要的）
    3. Base case 返回值（手动推演最底层那个 return 到底传给了谁）

6. 阿彪的开发感悟

初学者认为递归是“自己调用自己”，而高手认为递归是 “业务逻辑的数学抽象”。当你不再把递归当做一种“写法”，而是当做一种“状态转移的表达”时，你的算法段位就彻底上了一个台阶！
