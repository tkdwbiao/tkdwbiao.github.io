从入门到击败 99%：DFS 深度优先搜索万能模板

2026-03-16
阅读时长: 6 分钟

在算法面试中，DFS（Depth-First Search）是出镜率最高的算法之一。无论是岛屿问题、排列组合，还是复杂的路径搜索，其核心逻辑始终如一。

很多同学写 DFS 容易超时或写乱，本质上是因为没有掌握一套标准的“公式”。本文将总结三套最常用的 DFS 模板。

一、 DFS 的核心思维：剥洋葱

DFS 的本质就是递归。你可以把它想象成剥洋葱：

  1. 边界判定：洋葱剥完了吗？（递归终点）
  2. 当前处理：剥下一层皮。（处理当前节点）
  3. 继续扩散：去剥更深处的皮。（递归子节点）
  4. 回溯还原：把剥下的皮贴回去。（撤销选择，可选）

二、 模板 1：矩阵/网格类（沉岛式）

适用场景：岛屿数量、最大岛屿面积、图像渲染。 这类问题的特点是：不需要撤销操作，走过的地方直接“淹没”或标记。

这种写法在 Python 中效率最高，因为它减少了无效的函数调用开销。

def dfs(grid, r, c):
    # 1. 标记当前格子已访问（沉岛）
    grid[r][c] = "0"
    
    # 2. 定义方向：上下左右
    for dr, dc in [(0, 1), (0, -1), (1, 0), (-1, 0)]:
        nr, nc = r + dr, c + dc
        
        # 3. 先判断合法性，再进入递归（Python 性能优化关键）
        if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == "1":
            dfs(grid, nr, nc)

# 外层主循环
for r in range(rows):
    for c in range(cols):
        if grid[r][c] == "1":
            count += 1
            dfs(grid, r, c)

def backtrack(state, index, path):
    # 1. 递归终点（Base Case）
    if index == len(target):
        res.append(path[:]) # 注意：存入的是副本
        return True
        
    # 2. 尝试每一种选择
    for choice in possible_choices:
        if is_valid(choice):
            # 3. 做选择
            make_choice(choice)
            
            # 4. 进入下一层
            if backtrack(state, index + 1, path):
                return True # 如果只找一个答案，直接截断返回
            
            # 5. 撤销选择（回溯的关键！）
            undo_choice(choice)
            
    return False

四、 避坑指南：面试中的 3 个绝招

1. 永远先做“频次剪枝”

在执行 DFS 之前，先检查基础条件。例如《单词搜索》，如果棋盘里的字母总数都不够组成单词，直接返回 False。

1
2

if not Counter(word) <= Counter(c for row in board for c in row):
    return False

2. Python 的递归深度限制

Python 默认递归深度是 1000。如果矩阵很大，会报 RecursionError。 面试技巧：主动跟面试官提一句，如果需要处理超大规模数据，可以手动增加深度：

1
2

import sys
sys.setrecursionlimit(2000)

3. 传参还是传全局变量？

  • 推荐：将 index 作为参数传递（如 dfs(x, y, index)）。
  • 原因：函数调用栈会自动管理 index 的值，递归返回时 index 会自动回到上一层的状态，省去了手动执行 index -= 1 的痛苦。

五、 总结公式

DFS = 边界条件 + 剪枝优化 + 处理当前 + 递归四周 + 回溯还原

  • 沉岛类：不回溯，改 grid[i][j]。
  • 路径类：必回溯，改 visited 集合或改回原字符。

掌握这套模板，在面试中遇到搜索类题目时，你只需要像填空一样把逻辑填进去，Bug 率会降低 80% 以上！

class Solution:
    def numIslands(self, grid: List[List[str]]) -> int:
        # 边界情况处理：如果矩阵为空，直接返回 0
        if not grid: return 0
        
        # 获取矩阵的行数和列数
        rows, cols = len(grid), len(grid[0])
        count = 0

        # 定义 DFS 函数（沉岛逻辑）：其目的是将相连的陆地全部标记为水
        def sink(r, c):
            # 1. 标记当前格子为 "0"（表示已访问，防止递归回转或重复计算）
            # 这种原地修改的方式可以将空间复杂度从 O(M*N) 降至 O(1)（不计递归栈开销）
            grid[r][c] = "0"
            
            # 2. 遍历四个方向（右、左、下、上）
            for dr, dc in [(0, 1), (0, -1), (1, 0), (-1, 0)]:
                nr, nc = r + dr, c + dc
                
                # 3. 性能优化（LBYL）：先判断边界和条件，再发起递归调用
                # 这样可以减少 Python 昂贵的函数调用开销，提高执行速度
                if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == "1":
                    sink(nr, nc)

        # 遍历整个网格
        for i in range(rows):
            for j in range(cols):
                # 只要发现一块陆地 "1"，说明发现了一个新的岛屿
                if grid[i][j] == "1":
                    count += 1      # 岛屿计数加 1
                    sink(i, j)      # 立即使用 DFS 将与这块陆地相连的所有陆地“淹没”
                    
        return count

# --- 复杂度分析 (面试必考) ---
# 时间复杂度: O(M * N)，其中 M 和 N 分别为行数和列数。每个格子最多被访问两次（一次主循环，一次 sink）。
# 空间复杂度: O(M * N)，最坏情况下（全是陆地），递归深度会达到 M * N。

from collections import Counter

class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        rows, cols = len(board), len(board[0])
        
        # 优化：频次剪枝
        if not Counter(word) <= Counter(c for row in board for c in row):
            return False

        def dfs(r, c, index):
            if index == len(word) - 1: return True # 匹配到最后一个字母
            
            tmp, board[r][c] = board[r][c], "#" # 做选择
            for dr, dc in [(0,1), (0,-1), (1,0), (-1,0)]:
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols and board[nr][nc] == word[index+1]:
                    if dfs(nr, nc, index + 1): return True
            
            board[r][c] = tmp # 撤销选择
            return False

        for i in range(rows):
            for j in range(cols):
                if board[i][j] == word[0]:
                    if dfs(i, j, 0): return True
        return False

总结：两类 DFS 的关键区别

特性沉岛类（岛屿问题）回溯类（单词/排列）
状态修改grid[i][j] = "0"board[i][j] = "#"
是否回溯不需要（淹了就不用管了）必须还原（影响其他路径）
返回值通常不返回，或返回面积返回布尔值（找到没）或路径结果
核心目的遍历完整个连通块寻找一条满足条件的特定路径

面试贴士 [抱拳]

  1. 写之前先剪枝：如果遇到统计类问题，先用 Counter 扫一遍，如果不满足基本条件直接秒回 False。
  2. 递归深度：Python 默认深度 1000。如果面试题矩阵极大（如 500x500），记得跟面试官提一句 sys.setrecursionlimit。
  3. 坐标顺序：永远先判断 0 <= nx < rows，再访问 board[nx][ny]，否则会报 IndexError。
