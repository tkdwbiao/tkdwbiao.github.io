二分查找进阶：破解旋转数组的‘断层’密码

2026-03-23 23:00
阅读时长: 2 分钟

LeetCode 33. 搜索旋转排序数组 是二分查找的高阶变体。当原本有序的数组被切开并旋转，我们该如何保持 $O(\log n)$ 的效率？

1. 物理本质：总有一半是“正经人”

旋转数组（如 [4,5,6,7,0,1,2]）虽然整体乱了，但如果你在任何位置切一刀，左半部分或右半部分，必然有一个是排好序的。

  • 我们二分的目的不再是直接排除一半，而是先找到 “哪一半是有序的”。
  • 只有在有序的那一半，我们才能通过边界判断（nums[left] <= target < nums[mid]）来断定 target 在不在里面。

2. 算法决策树

假设中点是 mid：

情况 A：左半边有序 (nums[left] <= nums[mid])

  • 如果 target 刚好落在左半边的范围内，我们就把搜索范围锁死在左边（right = mid - 1）。
  • 否则，target 肯定在右半边那个“乱糟糟”的区域。

情况 B：右半边有序

  • 如果 target 落在右半边的范围内，我们锁死右边（left = mid + 1）。
  • 否则，去左边找。

“二分查找的精髓在于‘排除’。在旋转数组里，我们借用有序的一半来做排除法。”

3. 阿彪的细节笔记：为什么是 <=？

在判断 nums[left] <= nums[mid] 时，那个等号非常关键：

  • 当 left 和 mid 重合时（比如区间只有两个元素），我们默认左边是有序的。如果不加等号，逻辑会漏掉这种情况。

4. 复杂度分析

  • 时间复杂度：$O(\log n)$。虽然逻辑变复杂了，但每次依然排除了一半的元素。
  • 空间复杂度：$O(1)$。

阿彪的面试金句 [抱拳]

“面试官，旋转数组虽然失去了全局单调性，但依然保留了局部单调性。通过比较 nums[left] 和 nums[mid]，我们可以确定当前分裂出的哪一个区间是单调递增的。利用这个‘有序靠山’，我们可以通过常数次的边界比较，确定目标值是否在该区间内，从而维持了二分查找的对数级复杂度。”
