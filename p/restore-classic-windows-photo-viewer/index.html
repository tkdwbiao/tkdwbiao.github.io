极速回归：唤醒 Windows 10/11 隐藏的经典照片查看器

2026-03-21 13:30
阅读时长: 2 分钟

即使在 Windows 11 时代，许多开发者和摄影爱好者依然怀念 Windows 7 那个纯粹、极速的**“经典照片查看器”**。相比现代“照片”应用的加载转圈和繁琐界面，老版本胜在“秒开”。

其实，它从未离开过我们的系统，只是被微软“雪藏”了。今天阿彪带大家从底层原理出发，一键唤醒它。

1. 深度解析：它为什么“消失”了？

在 Windows 的文件关联体系中，每一个后缀名（如 .jpg）都需要指向一个处理程序。

核心组件：PhotoViewer.dll

经典照片查看器并不是一个独立的 .exe 程序，而是封装在 C:\Program Files\Windows Photo Viewer\PhotoViewer.dll 这个动态链接库里。

微软的“雪藏”手段

在 Windows 10/11 中，微软为了强推基于 UWP 架构的新版“照片”应用，在注册表中删除了大部分常见图片格式（JPG, PNG, BMP等）与 PhotoViewer.dll 的关联信息，仅保留了极少数（如 TIFF）的关联。

阿彪总结：我们要做的，不是安装软件，而是通过修改注册表，重新告诉系统：“这个 DLL 也能处理图片！”

2. 硬核操作：唤醒步骤

我们通过创建一个注册表脚本（.reg）来完成这一过程，这比手动一个一个改键值要安全、高效得多。

第一步：创建脚本

在桌面新建一个记事本文件，将以下代码完整复制进去：

Windows Registry Editor Version 5.00

; 指定 PhotoViewer 处理常见图片格式
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Photo Viewer\Capabilities\FileAssociations]
".jpg"="PhotoViewer.FileAssoc.Tiff"
".jpeg"="PhotoViewer.FileAssoc.Tiff"
".png"="PhotoViewer.FileAssoc.Tiff"
".bmp"="PhotoViewer.FileAssoc.Tiff"
".gif"="PhotoViewer.FileAssoc.Tiff"
".ico"="PhotoViewer.FileAssoc.Tiff"
".tif"="PhotoViewer.FileAssoc.Tiff"
".tiff"="PhotoViewer.FileAssoc.Tiff"

; 定义 PhotoViewer 的文件关联类
[HKEY_CLASSES_ROOT\PhotoViewer.FileAssoc.Tiff]
"EditFlags"=dword:00010000
"FriendlyTypeName"="@%ProgramFiles%\\Windows Photo Viewer\\PhotoViewer.dll,-305"

[HKEY_CLASSES_ROOT\PhotoViewer.FileAssoc.Tiff\DefaultIcon]
@="%SystemRoot%\\System32\\imageres.dll,-72"

[HKEY_CLASSES_ROOT\PhotoViewer.FileAssoc.Tiff\shell\open]
"MuiVerb"="@%ProgramFiles%\\Windows Photo Viewer\\PhotoViewer.dll,-304"

; 核心调用命令：利用 rundll32 调用 dll 中的函数
[HKEY_CLASSES_ROOT\PhotoViewer.FileAssoc.Tiff\shell\open\command]
@="rundll32.exe \"%ProgramFiles%\\Windows Photo Viewer\\PhotoViewer.dll\", ImageView_Fullscreen %1"

第二步：导入注册表

  1. 将记事本保存，重命名为 restore.reg（确保后缀名是 .reg 而不是 .txt）。
  2. 双击该文件，在弹出的系统警告中点击**“是”**。
  3. 看到“已成功添加”的提示后，唤醒动作就完成了。

3. 设置默认应用

此时，经典查看器已经具备了“接单”的资格，我们需要在系统层面正式任命它。

  1. 随意找一张图片（例如 test.jpg），点击右键。
  2. 选择 “打开方式” -> “选择其他应用”。
  3. 在弹出的列表中选择 “Windows 照片查看器”。
  4. 关键一步：勾选底部的 “始终使用此应用打开 .jpg 文件”。

4. 为什么阿彪推荐这么做？

  1. 零内存常驻：它只在打开图片时通过 rundll32.exe 调用，看完即走，不占用后台资源。
  2. 极速渲染：基于传统的 Win32 API 编写，渲染引擎极其简单直接。
  3. 无干扰体验：没有多余的修图功能、AI 分类或云同步提示，回归“查看图片”的本质。
