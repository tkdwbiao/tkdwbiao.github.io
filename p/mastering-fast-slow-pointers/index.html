指针的艺术：一篇文章带你吃透“快慢指针”四大黄金场景

2026-03-18 17:30
阅读时长: 3 分钟

在链表的世界里，由于没有下标，我们无法像数组那样直接访问中点或倒数某个位置。此时，快慢指针就像一把“瑞士军刀”，仅需 $O(1)$ 的空间，就能优雅地解决 80% 的链表难题。

什么是快慢指针？

快慢指针是指在遍历对象（通常是链表）时，使用两个移动速度不同的指针。

  • 快指针 (fast)：步长大（如每次走 2 步）。
  • 慢指针 (slow)：步小（如每次走 1 步）。

“利用步长差与时间差，在单向移动中找寻特定位置。”

场景一：寻找链表的中点 (Middle Point)

这是最基础的用法。当快指针走完 100% 的路程时，慢指针刚好走完 50%。

  • 核心代码：

slow = fast = head
while fast and fast.next:
    slow = slow.next       # 走 1 步
    fast = fast.next.next  # 走 2 步
return slow
  • 应用场景：876. 链表的中点、归并排序链表、判断回文链表。

场景二：判定环形链表 (Cycle Detection)

著名的“龟兔赛跑”算法。如果在操场跑步，速度快的人一定会“套圈”撞上速度慢的人。

  • 核心代码：
1
2
3
4
5
6
7

slow, fast = head, head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:
        return True # 相遇了，说明有环
return False
  • 应用场景：141. 环形链表、判断循环序列。

场景三：寻找环的入口 (Cycle Entrance)

这是一个带有数学魔力的进阶技巧。

  • 逻辑：
    1. 先按场景二找到相遇点。
    2. 将 fast 重新指向 head，slow 留在原地。
    3. 两人同时以步长 1 出发，再次相遇的地方就是环的入口。
  • 原理：基于数学推导 $a = c$（从头到入口的距离等于相遇点绕回入口的距离）。
  • 应用场景：142. 环形链表 II。

场景四：寻找倒数第 K 个节点 (K-th Node From End)

这是一种“等距”快慢指针。让快指针先跑，制造一个“时间差”。

  • 核心代码：

fast = head
# 1. 快指针先走 K 步
for _ in range(k):
    fast = fast.next
# 2. 慢指针从头出发，两人同步走
slow = head
while fast:
    slow = slow.next
    fast = fast.next
return slow # 当 fast 到底时，slow 刚好在倒数第 K 个
  • 应用场景：19. 删除链表的倒数第 N 个结点。

阿彪的面试话术 [抱拳]

“在处理链表问题时，如果需要获取某种‘位置关系’（如中点、距离末尾的距离、是否有环），我会优先考虑快慢指针。它的优势在于能够将原本需要 $O(n)$ 空间的哈希表方案优化为 $O(1)$ 的常数级空间，这在内存受限的系统开发中至关重要。”

阿彪复习贴士：

  1. 边界检查：写快慢指针时，while fast and fast.next 是标准写法，能有效防止 NoneType 报错。
  2. 逻辑记忆：
    • 中点/判圈：步长 1 比 2。
    • 倒数 K/环入口：步长 1 比 1（但有起始位置差）。
