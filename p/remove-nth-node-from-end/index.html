快慢指针进阶：如何优雅地切掉链表的倒数第 N 个节点？

2026-03-18 22:19
阅读时长: 2 分钟

在 LeetCode 19. 删除链表的倒数第 N 个结点 中，快慢指针是标配。但在实际书写中，有两种微小却有趣的写法差异。

1. 核心目标：寻找“前驱”

要删除一个节点，必须找到它的前一个节点。

  • 目标：倒数第 $n$ 个。
  • 我们要找：倒数第 $n+1$ 个。

“保持恒定的间距，是寻找倒数节点的终极逻辑。”

2. 两种写法对比

写法 A：阿彪的“越界停止法”

逻辑：让快指针领先 $n+1$ 个身位，直到快指针探出边界（None）。

slow = dummy
fast = dummy.next
for _ in range(n):
    fast = fast.next
while fast: # 探向 None
    slow, fast = slow.next, fast.next

写法 B：灵神的“末位停止法”

逻辑：让快指针领先 $n$ 个身位，直到快指针触碰到最后一个有效节点。

1
2
3
4
5

left = right = dummy
for _ in range(n):
    right = right.next
while right.next: # 触碰最后一个节点
    left, right = left.next, right.next

3. 为什么它们是等价的？（数学证明）

假设链表总长度为 $L$：

  • 写法 A：间距为 $n+1$，快指针走到底（$L+1$ 步），慢指针走了 $(L+1) - (n+1) = L-n$ 步。
  • 写法 B：间距为 $n$，快指针走到末尾（$L$ 步），慢指针走了 $L-n$ 步。

结论：两种写法的慢指针最终都停在了第 $L-n$ 个节点（即虚拟头节点向后移动 $L-n$ 次），这正是倒数第 $n$ 个节点的前驱！

4. 阿彪的避坑小贴士 [抱拳]

  1. 虚拟头节点 (Dummy Node)：这题必须要用！因为你可能要删除的是链表头（倒数第 $L$ 个）。没有 dummy，你会发现 slow 根本没地方站。
  2. 代码鲁棒性：虽然两种写法都对，但在面试中，写法 B（灵神版）稍微更稳健一点，因为它在 for 循环中 right 的移动次数更少，更不容易在极端 $n$ 值下触发空指针异常。
  3. 空间复杂度：始终保持 $O(1)$，这是双指针法的灵魂。
