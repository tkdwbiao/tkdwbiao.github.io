存、转、移、走：四步法彻底攻克反转链表

2026-03-17 21:30
阅读时长: 3 分钟

反转链表（LeetCode 206）是面试中最基础也最高频的题目之一。它的核心挑战在于：如何在调转指针方向的同时，不丢失后续节点的地址？

其实，只要记住四个字，你就能完美解决这个问题。

1. 核心逻辑：三指针同步

在反转过程中，我们需要维护三个位置：

  1. pre：当前节点的前驱（反转后，它将成为当前节点的 next）。
  2. cur：当前正在处理的节点。
  3. nxt：当前节点的后继（临时保存，防止断链）。

“存、转、移、走，步步为营。”

2. 四步口诀详解

在 while cur: 的循环体中，我们重复以下四个动作：

  1. 存：nxt = cur.next
    把“未来”存下来。因为下一步我们要修改 cur.next，如果不先存下来，后面的节点就找不到了。
  2. 转：cur.next = pre
    这是核心！ 把当前节点的指针指向它的前驱，完成反转。
  3. 移：pre = cur
    前任变现任。为了处理下一个节点，pre 指针需要向前移动一步。
  4. 走：cur = nxt
    现任变未来。cur 指针跳到刚才存好的 nxt 位置，继续处理。

class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        # 1. 基础情况处理
        if not head or not head.next:
            return head
        
        # 2. 初始化指针
        pre = None # 原来的头反转后指向 None
        cur = head
        
        # 3. 循环迭代
        while cur:
            # 第一步：存（临时保存下一步）
            nxt = cur.next
            
            # 第二步：转（反转指向）
            cur.next = pre
            
            # 第三步：移（更新前驱）
            pre = cur
            
            # 第四步：走（移动到下一个）
            cur = nxt
            
        # 4. 循环结束时，cur 为 None，pre 刚好停在原链表的尾部，即新链表的头
        return pre

4. 为什么最后返回 pre？

很多同学在面试时会犹豫最后返回 cur 还是 pre。

我们可以推演最后一步：

  • 当 cur 处理完最后一个有效节点时，它执行了 cur = nxt，此时 cur 变成了 None。
  • 而在此之前的一步 pre = cur，让 pre 停在了原链表的最后一个节点上。
  • 所以，pre 才是新链表的起始点。

5. 复杂度分析

  • 时间复杂度：$O(n)$。我们只需从头到尾扫描一遍链表。
  • 空间复杂度：$O(1)$。我们只额外使用了常数个指针变量（pre, cur, nxt），这正是面试官最想看到的优化结果。

阿彪的算法小结 [抱拳]

反转链表的关键在于**“先存后断”**。只要保证在断开当前连接之前，已经有指针指着下一个位置，你就永远不会丢失链表。

记住这个四步口诀：存、转、移、走。下次面试遇到它，直接秒杀！

复习要点：

  1. 初始化：pre 一定要设为 None，因为反转后，原先的第一个节点变成了最后一个，它的 next 必须指向 None。
  2. 循环条件：用 while cur: 最安全，它能自然处理空链表和单节点链表。
