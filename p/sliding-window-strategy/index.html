从逻辑到极致：滑动窗口攻克无重复最长子串

2026-03-17 15:30
阅读时长: 3 分钟

在解决字符串子串问题时，“滑动窗口” 是一种近乎直觉的优美解法。以 LeetCode 3. 无重复字符的最长子串为例，深度拆解如何从基础逻辑演进到极致优化的 $O(N)$ 解法。

1. 核心思路：窗口的收缩与扩张

这道题的本质是寻找满足“无重复”条件的最长区间。我们可以把这个区间想象成一个动态的窗口。

第一步：直觉逻辑——“一格一格挪”

想象你有一条贪吃蛇：

  1. 扩张：右边界 r 不停往后走，把新字符吃进来。
  2. 冲突：如果新吃的字符在窗口里已经有了，怎么办？
  3. 收缩：左边界 l 开始往后挪，每挪一格就“吐掉”一个字符，直到窗口里不再有那个重复的字符。
  4. 记录：在整个过程中，记下窗口达到的最大长度。

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        l = r = 0
        n = len(s)
        sub_s = ""
        res = 0
        
        while r < n:
            if s[r] in sub_s:
                sub_s = sub_s[1:] # 遇到重复，左侧切片
                l += 1
                continue
            sub_s += s[r]
            r += 1
            res = max(res, len(sub_s))
        return res

这种逻辑虽然正确，但由于左边界 l 是一格一格被动挪动的，效率还有提升空间。

2. 极致优化：哈希表“跳跃”法

为了让窗口移动得更快，我们引入哈希表（Map）。

第二步：进阶逻辑——“瞬间跳跃”

与其在遇到冲突时一格一格挪动 l，我们不如直接记住每个字符最后出现的下标。

  1. 记录：用一个哈希表存 {字符: 下标}。
  2. 跳跃：当右边界 r 发现重复字符时，左边界 l 不需要慢慢走。通过查表，l 可以直接 “跳” 到该重复字符上一次出现位置的下一个。
  3. 关键：跳跃时要确保 l 只能向右跳（使用 max 函数），不能往回走。

class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        # 哈希表存：{字符: 字符最后出现的下标}
        last_seen = {}
        l = 0  # 左指针
        res = 0
        
        # r 为右指针，负责遍历整个字符串
        for r in range(len(s)):
            char = s[r]
            
            # 1. 发现冲突：如果字符在哈希表中，说明可能重复
            if char in last_seen:
                # 2. 瞬间跳跃：左指针跳到重复位置的下一个
                # 必须用 max，防止左指针跳回窗口左侧以外
                l = max(l, last_seen[char] + 1)
            
            # 3. 记录/更新当前字符的最晚位置
            last_seen[char] = r
            
            # 4. 统计当前窗口长度，并更新最大值
            res = max(res, r - l + 1)
            
        return res

3. 复杂度深度分析

初始逻辑（切片/步进版）

  • 时间复杂度：$O(N^2)$ 或 $O(N \cdot K)$。因为在检查重复和字符串切片时，底层往往涉及一次 $O(K)$ 的额外遍历。
  • 空间复杂度：$O(K)$，用于存储临时子串。

优化逻辑（哈希表跳跃版）

  • 时间复杂度：$O(N)$。右指针遍历且仅遍历一次，左指针的跳转是 $O(1)$ 的。无论字符串多长，我们都只需要“线性”地扫描一遍。
  • 空间复杂度：$O(min(M, N))$。$M$ 是字符集的大小（比如 ASCII 码只有 128 位）。这是一种典型的空间换时间。

4. 算法避坑指南：$N$ 与 $N-1$

在面试中，指针什么时候该停，是判断一个开发者是否细心的标志。

场景一：使用 $N$ (即 len(s))

当你需要“每一个元素都踩到”时。

  • 用法：while i < len(s) 或 for i in range(len(s))。
  • 例子：滑动窗口、统计字符频次、求和。
  • 口诀：从头到尾，一个不留，统统用 $N$。

场景二：使用 $N-1$ (即 len(s) - 1)

当你需要“看邻居”时。

  • 用法：for i in range(len(s) - 1)。
  • 特征：代码里出现了 s[i+1]。
  • 原因：如果你不减 1，当 i 走到最后一个时，i+1 就会导致数组越界（IndexError）。
  • 例子：最长公共前缀、判断数组是否有序。

思路比代码更重要。希望这份关于滑动窗口的演进逻辑能帮你拿下心仪的 Offer！[握手]

复习要点：

  1. 跳跃逻辑：面试官问你“为什么用 max”，你要回答“为了防止左指针跳回到已经排除掉的重复区域”。
  2. 复杂度：一定要强调哈希表带来的 $O(1)$ 随机访问能力 如何把整体复杂度降到了线性级。
