贪心算法全攻略：如何在‘短视’中成就全局最优？

2026-03-21 16:10
阅读时长: 4 分钟

在算法的世界里，贪心算法（Greedy Algorithm）是一种极具诱惑力的思维方式。它要求我们在每一步选择中都采取当前状态下最好或最优的选择，从而希望导致结果是全局最优的。

今天阿彪带大家拆解贪心算法的底层逻辑、适用场景以及面试中的实战套路。

1. 贪心算法的核心哲学

贪心算法不从整体最优上加以考虑，它所做出的选择仅仅是局部最优选择。

“贪心法不做后悔药：一旦做出选择，绝不回溯。”

贪心 vs 动态规划 (DP)

  • 动态规划：会保存以前的运算结果，并根据以前的结果对当前进行选择，有“回头看”的过程。
  • 贪心：只根据当前信息做判断。它比 DP 效率更高（通常是 $O(n)$），但适用范围更窄。

2. 什么样的题可以用贪心？

要使用贪心，必须满足两个核心性质：

  1. 贪心选择性质：全局最优解可以通过一系列局部最优选择来达到。
  2. 最优子结构：问题的最优解包含其子问题的最优解。

阿彪的大白话判断法： 如果在做题时，你发现“最好的选择”非常明显，而且这个选择不会对后面的选择产生“负面限制”，那么这题大概率可以用贪心。

3. 贪心算法的三大经典模型

模型一：区间/覆盖类

核心思想：通过排序，让区间变得“有序”，从而方便挑选最远或最紧凑的路径。

  • 代表题：55. 跳跃游戏
  • 策略：每次都更新当前能跳到的最远边界，只要当前步子没断，就一直往前走。

模型二：分配类

核心思想：将资源优先分配给最容易满足的对象。

  • 代表题：455. 分发饼干
  • 策略：把最小的饼干分给胃口最小的孩子。这种“刚好满足”的策略能为后续节省资源。

模型三：反转/贪心转换类

核心思想：利用数值的特性（如正负号），在特定时刻进行反转。

  • 代表题：1005. K 次取反后最大化的数组和
  • 策略：优先反转绝对值最大的负数。

4. 实战模板

模型一：区间/覆盖类（寻找最远边界）

核心思想：不需要关心怎么跳过来的，只需要关心现在最远能跳到哪。

📝 例题：55. 跳跃游戏

题目描述：给定一个非负整数数组，你最初位于第一个下标。每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。

阿彪的贪心逻辑： 维护一个 max_reach 变量。遍历数组，每到一个新点，看能否刷新“最远地平线”。如果当前位置已经超过了 max_reach，说明此路不通。

class Solution:
    def canJump(self, nums: List[int]) -> bool:
        max_reach = 0 # 记录目前能触达的最远地平线
        
        for i, jump in enumerate(nums):
            # 1. 检查路是否断了
            if i > max_reach:
                return False
            
            # 2. 贪心选择：更新最远边界
            max_reach = max(max_reach, i + jump)
            
            # 3. 提前终止
            if max_reach >= len(nums) - 1:
                return True
        return False

模型二：数值转换类（最大化贡献度）

核心思想：利用数值特性（如正负号），优先处理对结果影响最大的元素。

📝 例题：1005. K 次取反后最大化的数组和

题目描述：给定一个整数数组和 $K$，你可以进行 $K$ 次翻转（正变负，负变正），求翻转后数组可能的最大和。

阿彪的两次贪心逻辑：

  1. 第一次贪心：按绝对值从大到小排序。优先翻转绝对值最大的负数（“负得最狠”的变正，贡献最大）。
  2. 第二次贪心：如果负数翻完了 $K$ 还有剩余（且为奇数），我们就挑全场绝对值最小的那个数（离 0 最近，损失最小）翻转一次。

class Solution:
    def largestSumAfterKNegations(self, nums: List[int], k: int) -> int:
        # 1. 核心技巧：按绝对值降序排列
        nums.sort(key=lambda x: abs(x), reverse=True)
        
        # 2. 第一次贪心：遇到负数且 K > 0，立即翻转
        for i in range(len(nums)):
            if nums[i] < 0 and k > 0:
                nums[i] *= -1
                k -= 1
        
        # 3. 第二次贪心：若 K 仍为奇数，翻转末尾绝对值最小的数
        if k % 2 == 1:
            nums[-1] *= -1
            
        return sum(nums)

5. 阿彪的做题技巧：如何证明贪心是对的？

在面试中，面试官最喜欢追问：“你凭什么说这样选最后一定能得到最优解？”

阿彪建议你不需要进行复杂的数学证明，掌握这两种面试话术：

  1. 反证法：假设我不选当前的局部最优，而是选了另一个。你会发现，选另一个并没有为后续带来任何好处，反而可能让路变窄了。
  2. 替换法：如果存在一个全局最优解不包含我这个局部最优选，我可以证明把那个选替换成我这个，结果依然是最优的甚至更好。

6. 阿彪的面试避坑笔记 [抱拳]

  • 先排序，后贪心：90% 的贪心题都需要先进行 sort()。如果一头雾水，试试按区间右端点或左端点排个序，灵感可能瞬间就来了。
  • 警惕 0 的陷阱：在路径类贪心中，0 往往是阻断贪心逻辑的元凶，需要特判。
  • 复杂度意识：贪心算法的时间复杂度通常由排序决定（$O(n \log n)$）或是一次遍历（$O(n)$），空间通常是 $O(1)$。

贪心算法不仅仅是一个算法，更是一种‘活在当下、追求极致’的思维方式。阿彪希望大家在刷题时能一眼识破贪心的陷阱，精准狙击最优解！
