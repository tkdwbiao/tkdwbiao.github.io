动态规划进阶：从‘经验主义’到‘最优抉择’的思想全书

2026-03-21 23:00
阅读时长: 4 分钟

在算法面试的下半场，只有掌握了动态规划（DP），才算拿到了大厂的入场券。今天阿彪不聊玄学，我们从思想、方法、套路、实战四个维度，彻底拆解 DP 的本质。

一、 动态规划的灵魂：三大核心思想

很多同学觉得 DP 难，是因为没转过这个弯：

  1. 重叠子问题（记忆化思想）： 一个大问题会被拆解成无数个小问题，而这些小问题会反复出现。比如斐波那契数列，算 f(5) 要算 f(4) 和 f(3)，算 f(4) 也要算 f(3)。 阿彪悟道：不要做重复劳动，算过一次就拿小本本（数组/哈希表）记下来。

  2. 最优子结构： 大问题的最优解，一定是由子问题的最优解推导出来的。 阿彪悟道：如果你能通过“昨天的最优表现”决定“今天的最优表现”，那这就是 DP。

  3. 状态转移（决策思想）： 这是 DP 的精髓。每一格 dp[i] 实际上代表了一个决策的结果。你要在“选”或“不选”、“跳”或“不跳”之间做出选择。

二、 阿彪的“五步断案法”

面对任何 DP 题，不要直接写代码，按这个顺序走：

  1. 定义状态（最难的一步）：dp[i] 到底代表什么？是“第 i 天的最大收益”？还是“凑齐 i 块钱的最少硬币数”？
  2. 推导方程：dp[i] 是怎么从 dp[i-1]、dp[i-2] 变过来的？
  3. 初始化边界：最简单的情况（如 dp[0]）是什么？
  4. 确定方向：是 $0 \rightarrow n$（正序）还是 $n \rightarrow 0$（倒序）？
  5. 举例打印：如果结果不对，把 dp 数组打印出来，看它是不是按你预想的数字增长。

三、 四大经典模型实战

1. 打家劫舍系列：决策的艺术

这类题目的思想是：“相邻不可选”。

  • 线性打家劫舍 (LC198)：dp[i] = max(dp[i-1], dp[i-2] + nums[i])。
  • 环形打家劫舍 (LC213)： 阿彪黑科技：既然第一间和最后一间不能同时抢，那就拆成两场比赛： 一场不抢第一间，一场不抢最后一间。最后取 max。
  • 树形打家劫舍 (LC337)： 阿彪进阶：每个节点返回两个值：[偷当前节点的最大收益, 不偷当前节点的最大收益]。

2. 背包问题系列：资源的极致利用

背包问题的核心在于：如何在有限的负重下，塞进价值最高的东西？

① 01 背包（每件仅一个）

思想：对于第 $i$ 件物品，我有两个选择：拿（消耗容量，增加价值）或者不拿。 阿彪注意：一维空间优化时，必须倒序遍历容量！因为要保证计算 dp[j] 时，引用的 dp[j-w] 是上一层（还没放这件物品）的数据。

for weight, value in items:
    for j in range(bag_weight, weight - 1, -1): # 倒序是灵魂
        dp[j] = max(dp[j], dp[j - weight] + value)

② 完全背包（每件无限个）

思想：可以一直拿。 阿彪注意：正序遍历容量。因为 dp[j] 依赖的 dp[j-w] 应该是“已经考虑过当前物品”后的状态。

2
3

for weight, value in items:
    for j in range(weight, bag_weight + 1): # 正序代表可重复
        dp[j] = max(dp[j], dp[j - weight] + value)

③ 多重背包（每件有固定数量 $k_i$）

思想：最暴力的办法是把 $k_i$ 个物品看成 $k_i$ 个独立的 01 背包。 高手写法：利用二进制拆分（1, 2, 4, 8…），可以把 $k_i$ 个物品组合成 $\log k_i$ 组，极大地优化效率。

4. 最长递增子序列 (LIS - LeetCode 300)

  • 状态：dp[i] 代表以第 $i$ 个数结尾的最长递增子序列长度。
  • 方程：dp[i] = max(dp[j] + 1)，其中 $j < i$ 且 nums[j] < nums[i]。

5. 最长公共子序列 (LCS - LeetCode 1143)

  • 状态：dp[i][j] 代表字符串 A 前 $i$ 个和 B 前 $j$ 个的最长公共子序列。
  • 方程：
    • 如果 A[i] == B[j]，则 dp[i][j] = dp[i-1][j-1] + 1
    • 否则，取 max(dp[i-1][j], dp[i][j-1])。

6. 路径规划系列：降维打击

  • 不同路径 (LC62)：dp[i][j] = dp[i-1][j] + dp[i][j-1]。 阿彪建议：所有的二维路径 DP 都可以优化成一维滚动数组。

四、 阿彪的高级 DP 锦囊

1. 空间压缩（滚动数组）

如果 dp[i] 只和 dp[i-1] 有关，那你根本不需要开 $O(n)$ 的数组，只需要两个变量 prev 和 cur 即可。这能把空间复杂度从 $O(n)$ 压到 $O(1)$，面试官看到会直接加分。

2. 方案数 vs 最值

  • 求最大/最小：通常初始化为 0 或正无穷，方程用 max() 或 min()。
  • 求方案总数：通常初始化为 0，基础位置为 1，方程用 sum()。

3. 阿彪的心理建设

如果一道题你半小时没思路，不要死磕状态转移方程。 先写一个纯递归（DFS），然后观察：是不是有很多重复计算？如果是，加个 Memo（备忘录）。 恭喜你！记忆化搜索其实就是动态规划。

阿彪小结 [抱拳]

动态规划不是一种算法，而是一种寻找最优路径的哲学。 不要害怕方程，去画图、去模拟、去观察数字是怎么一点点填满那个表格的。

“所谓的 DP 高手，不过是比你多画了几个表格，多拆了几次状态而已。”
