最大子数组和：为什么我的 O(n²) 动态规划会超时？

2026-03-22 23:10
阅读时长: 3 分钟

在 LeetCode 53. 最大子数组和 的战场上，很多同学（包括最初的我）都会掉进 $O(n^2)$ 的泥潭。今天阿彪带大家深度复盘：为什么看似合理的 DP 会超时？而线性 $O(n)$ 甚至极致的 $O(1)$ 空间算法又是如何诞生的？

1. 阿彪的初版：沉重的全量记录

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        n = len(nums)
        res = nums[0]
        dp = [[0]*n for i in range(n)] # 试图记录起点为 j，终点为 i 的所有可能和
        dp[0][0] = nums[0]
        for i in range(1, n):
            for j in range(0, i + 1):
                dp[i][j] = dp[i-1][j] + nums[i]
                res = max(dp[i][j], res)
        return res

📉 深度分析：为什么它们必死无疑？

  1. 计算量冗余：这两份代码本质都在枚举所有子数组。对于 $n=10^5$ 的数据，运算量达到了 50 亿次，远超 LeetCode 1 秒钟处理约 $10^8$ 次的上限。
  2. 结论：记录“每一个区间”是低效的，我们要学会“只记录最优决策”。

2. 核心突破：从“全量”到“以 $i$ 结尾”

我们要找的是最大和，站在任何一个数字 $x$ 的面前，我们只需要问自己一个问题：

“前面的积累对我是助力，还是累赘？”

  • 助力：如果前面的和大于 0，那它能让我变得更大，我要带着它。
  • 累赘：如果前面的和小于等于 0，它只会拖累我，我直接从我自己重新开始。

3. 进阶优化：$O(n)$ 线性动态规划

通过定义 dp[i] 为 “以第 i 个元素结尾的最大子数组和”，我们成功将逻辑降维。

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        n = len(nums)
        dp = [0] * n
        dp[0] = nums[0]
        for i in range(1, n):
            # 决策：前面的 dp[i-1] 若是负数，则舍弃取 0
            dp[i] = max(dp[i-1], 0) + nums[i]
        return max(dp)

4. 终极优化：$O(1)$ 空间 Kadane 算法

观察发现，dp[i] 只取决于 dp[i-1]。我们连数组都不用开，只用一个变量滚动更新，这就是大名鼎鼎的 Kadane 算法。

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        cur_sum = 0
        max_sum = nums[0] # 永远用第一个元素初始化，防止全负数陷阱
        
        for x in nums:
            # 决策：是把前面的带上，还是从自己起步？
            cur_sum = max(cur_sum + x, x)
            # 更新全局见过的最高峰
            max_sum = max(max_sum, cur_sum)
            
        return max_sum

6. 阿彪的面试避坑笔记 [抱拳]

  1. 别被 DP 两个字吓住：很多时候我们不需要巨大的表格。如果当前状态只和“前一格”有关，**“滚动变量”**是首选。
  2. 初始化陷阱：在求最大值时，不要初始化 max_sum = 0。如果数组全是负数 [-2, -1]，答案应该是 -1。务必用 nums[0] 初始化。
  3. “以 i 结尾”的魔力：这是解决序列类题目（如最长递增子序列、最长连续序列）的通用金钥匙。

从 $O(n^2)$ 到 $O(n)$，不仅是算法的优化，更是思维的断舍离。阿彪祝大家早日 AC，拿到 Offer！
