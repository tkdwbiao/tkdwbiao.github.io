搜索二维矩阵：从‘两次二分’到‘虚拟一维’的逻辑升华

2026-03-23 21:40
阅读时长: 2 分钟

在 LeetCode 74. 搜索二维矩阵 中，矩阵的每一行都是有序的，且每一行的第一个整数大于前一行的最后一个整数。

这种全局有序性给了我们很大的优化空间。今天阿彪分享两种二分查找的实战解法。

方案一：分而治之（两次二分查找）

阿彪的思想：既然矩阵行与行之间也有序，那我们可以分两步走：

  1. 锁定行：对每一行的最后一个元素进行二分查找，确定 target 可能落在第几行。
  2. 锁定列：在确定的那一行里，再进行一次标准二分查找。

class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        # 1. 提取每一行的最后一个元素，构成一个列向量
        last_elements = [row[-1] for row in matrix]
        
        def binary_search(nums, t):
            l, r = 0, len(nums) - 1
            while l <= r:
                mid = l + (r - l) // 2
                if nums[mid] < t: l = mid + 1
                else: r = mid - 1
            return l

        # 2. 第一次二分：找 target 可能存在的行
        row_idx = binary_search(last_elements, target)
        
        # 边界检查：如果 target 比全场都大，直接返回 False
        if row_idx >= len(matrix):
            return False
            
        # 3. 第二次二分：在该行内查找
        col_idx = binary_search(matrix[row_idx], target)
        
        # 最终判断该位置的值是否匹配
        if col_idx < len(matrix[0]) and matrix[row_idx][col_idx] == target:
            return True
        return False
  • 优点：逻辑分明，符合人类第一直觉。
  • 缺点：需要 $O(M)$ 的额外空间来存储 last_elements，且代码较为繁琐。

方案二：降维打击（虚拟一维化）

阿彪的思想：如果把这个矩阵的所有行首尾相连，它其实就是一个长度为 $M \times N$ 的超长有序数组。我们不需要物理上真的去转换它，只需要在逻辑上映射坐标。

核心魔术：坐标转换公式

假设矩阵总列数为 n，一维索引为 idx：

  • 行坐标：r = idx // n
  • 列坐标：c = idx % n

class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        if not matrix or not matrix[0]: return False
        
        m, n = len(matrix), len(matrix[0])
        left, right = 0, m * n - 1
        
        while left <= right:
            # 取一维的中点
            mid = left + (right - left) // 2
            # 利用公式【逆向还原】二维坐标
            val = matrix[mid // n][mid % n]
            
            if val == target:
                return True
            elif val < target:
                left = mid + 1
            else:
                right = mid - 1
        return False
  • 优点：极致优雅。只需一次二分，空间复杂度压缩到 $O(1)$，没有繁琐的边界判断。

“面试官，这道题的矩阵满足全局单调性，因此具备了‘逻辑一维化’的条件。我通过 index // cols 和 index % cols 将一维二分的索引实时映射回二维坐标，从而在 $O(\log(MN))$ 的时间内以 $O(1)$ 的空间消耗完成搜索。这种‘坐标投影’的思想不仅代码简洁，也避免了多次处理边界的风险。”
